## Purpose

Deliver the Get Found foundation cleanup consistently: profile access, business facts, visibility readiness, review link, proof snapshot, and client-safe completion summary.

## Covered Master Map Rows

- GBP audit
- Google AI Search readiness audit
- GBP category and service review
- Hours, phone, website, and description review
- Website/profile fact sync
- Review link capture and test
- Before/after visibility snapshot
- Get Found completion audit
- Get Found delivery email

## Trigger

Client buys Get Found or Get Found work starts inside a higher plan.

## Roles

| Role | Responsibility |
|---|---|
| Profile Manager | Performs GBP/profile/fact checks and recommends fixes |
| Systems Director | Supports website/crawlability/schema handoffs |
| Reporter | Captures baseline and before/after proof |
| Auditor | Checks completion proof and claims |
| Account Manager | Sends client-safe completion summary |

## Hard Rules

- Do not guarantee rankings, calls, reviews, AI Overview placement, or Google actions.
- Do not make public profile edits without correct access and approval path.
- Do not invent business facts. If uncertain, ask the client.
- Do not mark Get Found done until Auditor approves proof.

## Procedure

1. Confirm authorized scope.
   - Verify client, plan, payment/handoff, and any approved custom scope.
   - Confirm GBP access status and client hub record.

2. Run baseline visibility check.
   - Reporter records current public state: website, GBP, business facts, reviews, photos, services, profile completeness, crawlability clues, and competitor context when appropriate.

3. Audit GBP foundation.
   - Check name, category, services, hours, phone, website, description, service area/address, photos, review link, and profile completeness.
   - Record fix plan before editing.

4. Check website/profile fact sync.
   - Compare GBP facts against website and client record.
   - Flag mismatches for correction or client clarification.

5. Check AI/Search readiness basics.
   - Confirm public crawlable facts, clear services/locations, helpful proof, review freshness, and obvious missing trust signals.
   - Merchant Center/UCP only applies if the client sells eligible products, bookings, food delivery, hospitality, or ecommerce.

6. Complete approved fixes.
   - Make only in-scope, approved updates.
   - Capture before/after proof for material changes.

7. Capture review link.
   - Generate/test the Google review link.
   - Hand off to Reviews Manager for request path setup if review service is included.

8. Auditor completion check.
   - Auditor confirms scope, proof, profile identity, claims, and client-facing summary.
   - If proof is missing, keep status in progress.

9. Send delivery summary.
   - Account Manager sends a short summary of what changed, what is waiting, and what comes next.
   - Update client hub/report center.

## Required Proof

- Baseline notes/report
- GBP/profile access verification
- Fix checklist
- Before/after screenshots or public links
- Review link test
- Auditor approval
- Delivery email/hub update

## Failure Or Blocker Handling

- GBP access missing: return to SOP 005.
- Business fact conflict: ask client before editing.
- Website issue needs dev work: route to Systems Director.
- Public edit risk: route to Auditor/Manager.
- Client asks for out-of-scope SEO: Sales Manager decides upgrade/custom scope.
