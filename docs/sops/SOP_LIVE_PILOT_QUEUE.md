## Pilot Rule

A live pilot must use the first real low-risk event, recurring cycle, controlled non-production test, or Mike-approved investigation that matches the SOP trigger. The pilot must capture proof, then Auditor decides pass, hold, block, or needs update.

## Evidence Template

Use `docs/sops/TEMPLATE-sop-live-pilot-evidence.md` for each live pilot.

## Do Not Force These Events

- Do not trigger refunds, disputes, complaints, incidents, negative reviews, production deploys, or payment-risk workflows just to test an SOP.
- Use the first natural event, or a safe non-production test where the SOP allows it.
- Do not run live Smartlead sends, review sends, SMS sends, public profile edits, HighLevel AI toggles, or agentic checkout work without the required approvals.
