# Friend Test Client Pilot

Status: candidate identified
Owner: Manager/Coach
Created: 2026-05-27
Purpose: use a trusted friend as a low-risk test client to live-pilot GMF onboarding, visibility reporting, Google Business Profile access, Get Found fulfillment, review path setup, and client recap workflows.

## Pilot Principle

Treat this like a real client workflow, but label it clearly as a test/non-billing pilot so financial, sales, and promise tracking stay clean.

Do not fake payment, reviews, customer consent, Google access, or public results. Use real proof only.

## Recommended Pilot Path

1. Manual client start approval
2. Client intake and consent
3. Client record, slug, folder, and hub shell
4. Magic-link access test
5. Welcome email and first asks
6. Google Business Profile access request
7. Baseline visibility report
8. GBP access verification
9. GBP audit and Google AI Search readiness audit
10. Get Found fixes/recommendations
11. Before/after visibility snapshot
12. Get Found completion audit
13. Client-safe delivery email
14. Review link capture and first review request path setup
15. Monthly/client recap dry-to-live pilot when enough activity exists

## Intake Needed From Mike/Friend

Minimum:

- Friend/business owner name
- Business name
- Business website
- Primary service category
- Service area or public address rules
- Business phone
- Business email/contact
- Current Google Business Profile link, if it exists
- Whether GMF can be added as a Manager on the GBP
- Whether this is a test/non-billing pilot
- What the friend is comfortable letting GMF change publicly

Helpful:

- Top 3 services
- Main city/service area
- Top 3 competitors
- Current customer-list availability
- Current review link, if known
- Photos/logo
- Any sensitive categories or claims to avoid

## Friend Consent Language

```text
You are helping us test the GetMeFound client workflow. We will treat your business carefully and only make public changes you approve. We will not ask for your Google password. If Google Business Profile access is needed, you will add our business email as a Manager so you stay in control.
```

## Public Change Rule

Before any public GBP/site/profile edit:

1. Profile Manager writes proposed edit.
2. Auditor checks claim and profile risk.
3. Friend/client approves the edit.
4. Profile Manager makes the edit.
5. Reporter captures before/after proof.

## Review Request Rule

Do not send review requests to the friend's customers unless:

- The friend confirms they are real recent customers.
- The friend has permission/relationship to contact them.
- Suppression and consent checks pass.
- Proof preview is approved.
- Manager approves the live send.
