# SOP 008 - Monthly Client Recap

## Purpose

Create a client-safe monthly recap that proves work, explains movement, names blockers, and recommends the next useful action without overclaiming.

## Covered Master Map Rows

- Monthly Stay Found recap
- Monthly recap audit
- Monthly recap send
- Client report artifact generation
- Report download/archive
- Search Console and AI visibility signal review when included

## Trigger

Month-end recap cycle, or Manager requests a client recap.

## Roles

| Role | Responsibility |
|---|---|
| Reporter | Builds recap and report artifact |
| Profile Manager | Supplies GBP/profile activity and drift notes |
| Reviews Manager | Supplies review request/review activity |
| Auditor | Checks claims, proof, and client-safe wording |
| Account Manager | Sends recap and handles client response |

## Hard Rules

- Do not report unsupported movement as success.
- Do not turn the recap into a hard upsell.
- Do not include private internal notes.
- Do not show prospect-only locked sections to paying clients as the main sales device.
- Upgrade opportunity appears only as one plain next opportunity when proof supports it.

## Procedure

1. Pull client context.
   - Confirm plan, month, active services, blockers, previous recap, and client hub link.

2. Gather proof.
   - Review requests sent, clicks/replies if available, new reviews, GBP posts, profile edits, baseline/current visibility observations, report artifacts, and blockers.

3. Build recap.
   - Sections: what was done, what changed, proof, blockers/waiting on client, next action.
   - Keep it plain-English and owner-readable.

4. Add visibility notes.
   - Include Search Console/AI/local observations only when available and supportable.
   - No ranking guarantees or AI placement claims.

5. Audit.
   - Auditor checks every claim against proof.
   - Remove unsupported claims, inflated language, or risky recommendations.

6. Send/update.
   - Account Manager sends recap or updates client hub.
   - Report download/archive link is saved.

7. Route next action.
   - Blockers go to Account Manager/Manager.
   - Upgrade signals go to Sales Manager for timing decision.
   - System/report issues go to Systems Director/Reporter.

## Required Proof

- Recap artifact
- Proof links/screenshots/logs
- Auditor approval
- Sent email or hub update
- Next action owner

## Failure Or Blocker Handling

- Missing proof: report the absence plainly or hold claim.
- Client upset/risk: Manager handles before recap send.
- Billing issue: hold upsell language and route to Manager/Mike if needed.
