# SOP 042 - Custom Promise Approval

Status: Drafted
Version: 0.2
Owner: Sales Manager
Reviewer: Auditor
Approver: Manager; Mike for pricing/spend/legal/reputation risk
Effective date: Set when Active
Next review: Set when Active
Source of truth: `docs/sops/SOP-042-custom-promise-approval.md`

## Purpose

Prevent sales promises from creating delivery, legal, billing, reputation, or tool-cost debt.

## Trigger

Prospect or client asks for nonstandard scope, deadline, discount, guarantee, integration, AI feature, reporting, checkout/payment behavior, or special handling.

## Hard Rules

- Sales Rep cannot approve custom promises.
- No ranking, review, revenue, Google, AI Search, or timeline guarantees.
- No HighLevel AI feature enablement without explicit Mike authorization.
- Any pricing/spend/refund/legal/reputation/payment exception routes to Mike.

## Procedure

1. Capture exact requested promise.
2. Classify risk: scope, price, deadline, legal/privacy, reputation, tool spend, access, AI feature, payment/checkout.
3. Check whether existing offer covers it.
4. If in scope, Sales Manager writes approved wording.
5. If out of scope, Manager/Mike decision required depending on risk.
6. Log approved/declined wording before prospect/client response.

## Required Proof

- Request text
- Risk classification
- Decision
- Approved wording or decline note
- Approver

## Failure Or Blocker Handling

- Promise already made without approval: escalate to Manager immediately and audit client/prospect message.
- Ambiguous request: ask clarifying question before promising.
- Risky request: hold response until decision.
